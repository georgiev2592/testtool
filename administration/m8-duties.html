Peter Georgiev I worked on the TestManager implementation. More specifically on Create Test, Edit Test and Delete Test. After the user logs in he/she clicks on the TestManager button from the home screen. The system shows a drop down menu with two options: Create Test, and Manage Tests. If the user clicks on the Create Test option, the system open a new dialog which covers the basic functionality of Creating a test. When the user clicks on the "Create" button, the system make a call to the model and it displays a message in the terminal saying that the CreateTest method has been called. Similarly, if the user click on the Manage Tests option, the system open a new dialog, where the user can search our database for a particular test. If the there is not test that matches user's search criteria the system responds with an alert in the form of a popup. If the system performs a successful search, the user selects one of the tests and clicks on the "Edit" or "Delete" button, the system calls the appropriate model method that displays in the terminal a message saying which method has been called.
Note: In order to successfully search for a test, the user have to search for "final exam".

The model class with Spect specifications is TestManager.java (path: /implementation/source/java/testtool/model/testmanager/TestManager.java). The generated Spest test is in TestManagerTest.java (path: /implementation/spest/testtool/model/testmanager/TestManagerTest.java).
